### Bit Depth 정밀도 

- 이미지가 얼마나 많은 색상을 표현할 수 있느냐를 의미 
- 정밀도가 높을 수록 데이터의 폭이 넓어지고 더 자연스로운 이미지로 표시 

### 비트 표현

- 색상을 표현할 때 적어도 8비트 ( 256 = 2^8 ) 이어야 유의미한 데이터를 얻게 되어 색상을 표현 할 수 있음
- 8비트 정밀도를 사용할 때 흑백 색상을 원활하게 표현할 수 있으며, 주로 `그레이스케일(Grayscale)` 메서드를 많이 사용 
- 1비트 이미지는 이진화(BINARY)이미지 가 되며 4ㅂ지트 이미지는 저화질 이미지가 된다. 

OpenCV 기준으로 Python(`cv2` + `numpy`)과 C#(`OpenCvSharp`)의 정밀도(비트 깊이) 형식을 같이 보면 아래처럼 대응됩니다.

| OpenCV Depth | Python dtype (numpy) | C# MatType 예시 | 값 범위(대략) | 주 용도 |
|---|---|---|---|---|
| `CV_8U` | `np.uint8` | `CV_8UC1`, `CV_8UC3` | 0 ~ 255 | 일반 이미지(BGR/Gray) 기본 |
| `CV_8S` | `np.int8` | `CV_8SC1`, `CV_8SC3` | -128 ~ 127 | 특수 연산/필터 결과 |
| `CV_16U` | `np.uint16` | `CV_16UC1`, `CV_16UC3` | 0 ~ 65535 | 깊이맵, HDR, 센서 데이터 |
| `CV_16S` | `np.int16` | `CV_16SC1`, `CV_16SC3` | -32768 ~ 32767 | Sobel/Laplacian 중간 결과 |
| `CV_32S` | `np.int32` | `CV_32SC1`, `CV_32SC3` | 약 $\pm2.1\times10^9$ | 라벨링/정수 연산 결과 |
| `CV_32F` | `np.float32` | `CV_32FC1`, `CV_32FC3` | 실수 | 정규화, 모델 입력, 정밀 연산 |
| `CV_64F` | `np.float64` | `CV_64FC1`, `CV_64FC3` | 실수(고정밀) | 수치해석/고정밀 계산 |

채널 표기 규칙도 동일합니다.

| 의미 | Python 예 | C# 예 |
|---|---|---|
| 1채널(Gray) | `dtype=np.uint8`, shape `(h,w)` | `MatType.CV_8UC1` |
| 3채널(BGR) | `dtype=np.uint8`, shape `(h,w,3)` | `MatType.CV_8UC3` |
| 4채널(BGRA) | `dtype=np.uint8`, shape `(h,w,4)` | `MatType.CV_8UC4` |

핵심만 정리하면:
1. `8U`는 화면 표시용 표준 이미지 타입
2. 필터 중간 결과는 `16S`/`32F`를 자주 씀
3. Python은 `dtype`, C#은 `MatType`으로 같은 개념을 표현합니다.

`CV_<n-bit><S|U|F|>C<채널수>`

### 채널

- 채널은 그래픽스 이미지의 색상 정보를 담고 있다. `red`, `green`, `blue`와 추가로 `Alpha`로 구성 된다.
- 색상을 표시할 때는 3~4 채널의 값을 선택 
- 흑백을 표현할 때는 1채널을 사용 
- 해당 색에 가까워 질 수록 흰색(255), 멀 수록 흑백(0)으로 표시 된다. 
  - 즉 R,G,B별로 나누었을때 R이미지에서 red color가 흰색으로 표시, 
  - G의 경우 green 컬러가 흰색과 가까운 표시
  - B의 경우 blue 컬러가 흰색과 가까운 표시 

### 관심 영역 

- OpenCV에서는 주요 데이터로 N차원 배열을 사용 
- OpenCV의 구성요소중 하나의 데이터가 커질 때 마다 기하급수적으로 처리해야할 데이터가 늘어난다. 
- 이를 방지하고자 관심영역을 사용해 불필요한 데이터의 양을 줄인다. 
- 이 때 관심 영역을 Region Of Interest ( ROI )라고 부른다. 
- 이미지를 처리할 때 객체를 탐지하거나 검출하는 영역을 명확하게 관심 영역이라 지정할 수 있다. 

### 관심 채널 

- 관심 채널은 영역과 동일하게 데이터의 형태를 변형하는 것이 아니라 특정 영역을 불러와 연산량의 감소와 정확도를 향상시키는데 사용 
- COI(Channel Of Interest )라고 부르며, 이미지를 처리할 떄 특정 채널을 사용해 연산을 진행하는 경우 관심 채널이라 할 수 있다. 

### 히스토 그램 

OpenCV에서 히스토그램은 이미지의 픽셀 값 분포를 막대그래프로 나타낸 것입니다.

간단히 보면:

1. x축: 픽셀 값(예: Gray는 0~255)
2. y축: 해당 값이 이미지에 몇 개 있는지(빈도)

왜 쓰냐면:

1. 이미지가 전체적으로 밝은지/어두운지 파악
2. 대비(contrast)가 낮은지 높은지 판단
3. 임계값(Threshold) 설정이나 평활화(Equalization) 전처리에 활용

예시 해석:

1. 왼쪽(0 근처)에 몰리면 어두운 이미지
2. 오른쪽(255 근처)에 몰리면 밝은 이미지
3. 가운데 좁게 몰리면 대비가 낮음
4. 전체 구간에 고르게 퍼지면 대비가 비교적 좋음

컬러 이미지는 보통 B, G, R 채널별로 히스토그램을 따로 봅니다.  
그레이스케일은 1개 히스토그램만 보면 됩니다.
