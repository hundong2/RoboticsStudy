# 12. 캡스톤과 전문가 수준 평가

캡스톤 목표는 "명령을 받으면 로봇이 환경을 인식하고, 안전하게 이동하거나 조작하고, 실패 시 원인을 기록하는 시스템"을 설계하고 시연하는 것입니다.

## 추천 주제

자율 점검 로봇:

1. RViz 또는 웹 UI에서 목표 구역을 선택합니다.
2. Nav2가 목표 구역까지 이동합니다.
3. 카메라 노드가 이미지를 수집합니다.
4. perception 노드가 점검 대상을 탐지합니다.
5. 결과를 topic, log, rosbag2, 리포트로 남깁니다.
6. 실패하면 recovery behavior를 실행하고 원인을 기록합니다.

## 제출 산출물

- 요구사항 문서
- ROS 그래프 다이어그램
- topic/service/action/interface 목록
- TF tree
- launch 파일
- rosbag2 샘플
- 테스트 계획과 결과
- 실패 사례 분석
- 배포판과 RMW 선택 근거

## 전문가 수준 체크리스트

- [ ] 새 노드가 시스템 전체 latency에 미치는 영향을 측정했다.
- [ ] message type, frame id, QoS를 문서화했다.
- [ ] 시뮬레이션과 실제 로봇 차이를 리스크로 관리했다.
- [ ] 최소 1개 실패를 rosbag2로 재현했다.
- [ ] Nav2 또는 MoveIt2 설정 변경 이유를 설명했다.
- [ ] 최신 ROS 2 기능을 무작정 쓰지 않고 프로젝트 기준으로 평가했다.

## 합격 기준

전문가 수준은 모든 것을 외우는 상태가 아닙니다. 문제를 구조화하고, 도구로 관찰하고, 근거를 남기며, 안전하게 바꿀 수 있는 상태입니다.
